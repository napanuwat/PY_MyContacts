แนวทาง C — ข้อสรุป
ทุก field (ทั้ง Info และ Notes) trigger edit mode เมื่อคลิก
Banner ส้มโผล่ทุกครั้งที่เริ่มแก้ — รู้ตัวชัดว่าอยู่ใน edit mode
Notes tab ก็มี banner → อาจรู้สึก noisy เวลาพิมพ์ note บ่อยๆ
เหมาะถ้า info และ notes สำคัญเท่ากัน ต้องการ guard ชัดเจน
แนวทาง A+C — ข้อสรุป (แนะนำ)
Info tab = click-to-edit แบบ C (banner โผล่เมื่อคลิก info field)
Notes tab = always editable, autosave เงียบๆ — ไม่มี banner เลย
ตรงกับ use case หลัก: "เปิดมาบันทึก note → ปิด" ทำได้ทันที
เหมาะสำหรับ Test Manager ที่ update note บ่อยกว่าแก้ info contact
💡 สังเกต: ใน column ขวา (A+C) — เมื่อเปิด Notes tab จะพิมพ์ได้ทันทีโดยไม่ต้องทำอะไร แต่ถ้าต้องการแก้ Email ต้องคลิกที่ email field ก่อน (banner จะโผล่เฉพาะ Info tab)
